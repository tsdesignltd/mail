# MailDeck 📮

mac の「メール」(Mail.app) と連携して動く、ローカルWebアプリのメール整理ツールです。

- **迷惑メールの自動判定** — ルールベースで判定理由つきでリストアップし、ワンクリックで各アカウントの迷惑メールフォルダへ仕訳
- **メッセンジャー風UI** — 差出人ごとにメールを会話形式で閲覧。相手の受信は左、**自分の返信は右**に並ぶタイムライン
- **差出人ごとの同期** — 会話を開くと、その相手との受信＋送信の全履歴を全アカウント横断で取得
- **メールで開く** — 各メールの件名末尾の封筒アイコン ✉️ から Mail.app で開ける。**本文を開くと Mail.app 側でも既読になる**
- **差分同期** — 2回目以降は新着だけ取得して高速。**1時間ごとの自動同期**（ON/OFF）も可
- **よく使う相手の自動学習** — 受信頻度・返信履歴・直近性から重要な差出人を判定し、自動でリスト上部に追加
- **学習するブロック/信頼リスト** — 仕訳・「問題なし」操作が次回以降の判定に反映される

**🧪 デモ (サンプルデータ)**: https://tsdesignltd.github.io/mail/
GitHub Pages は静的ホスティングのため、デモは架空のサンプルデータで動きます。
実際のメール整理は下記の手順でローカル実行してください。

## 必要なもの

- macOS(Mail.app にメールアカウントが設定済みであること)
- それ以外は不要 — macOS 標準の Python 3 だけで動きます

## セットアップ(どの Mac でも同じ手順)

ターミナルで:

```sh
git clone https://github.com/tsdesignltd/mail.git
cd mail
```

起動は次のどちらでも:

- Finder で **`start.command` をダブルクリック**(Mail.app とブラウザも自動で開きます)
- またはターミナルで `python3 server.py` → ブラウザで http://localhost:8765

初回起動時に **「"ターミナル"が"メール"を操作しようとしています」** という
macOS の許可ダイアログが出るので「OK」を押してください。
その後、画面右上の「同期」を押すとメールの取得が始まります。

更新を取り込むときは `git pull` するだけです。

## 同期のしくみ

- **初回**は過去1ヶ月分をまとめて取得します（メールが多いアカウントは時間がかかります）。
- **2回目以降**は前回からの**新着だけ**を取得する「差分同期」で高速です。
- **会話を開く**と、その相手の受信＋送信の全履歴を自動で取得します（送信＝右側の表示や本文もこの時に揃います）。
- 設定の「同期の方式」から「**全体を再取得**」でキャッシュを作り直せます（表示がおかしい時用）。

### 自動同期(1時間ごと)

メニューバー、または設定の「同期の方式」で**自動同期をON**にすると、MailDeck を開いている間、
1時間ごとに自動で差分同期します（既定はOFF）。

### 既読について

- 会話で**メールの本文を開くと、Mail.app 側でも既読**になります。
- 一覧の各差出人行の「✓ 既読」ボタンは、**MailDeck の表示上のバッジだけ**を既読にします
  （Mail.app は変更しません。「全体を再取得」で実際の状態に戻ります）。

## 迷惑メールの仕訳先

迷惑候補タブでの仕訳も、会話画面の「⛔ ブロック」も、対象メールを
**各アカウント自身の迷惑メールフォルダ**(Gmailなら「迷惑メール」、iCloudなら
「Junk」)へ移動します。Gmail などは IMAP でサーバーにも反映されるため、
メールサービス側の迷惑メール学習にも効きます。メールは削除されないので、
誤判定時は迷惑メールフォルダから受信トレイへ手で戻せます。

## AI判定(オプション)

Claude Code CLI (`claude` コマンド) がインストールされている場合、
グレーゾーンのメールを「🤖 グレーゾーンをAI判定」ボタンでまとめて判定できます。
CLI が無い場合はボタンが表示されません(手動で「迷惑」/「問題なし」を選択)。

## データの保存場所とプライバシー

- `data/settings.json` — 信頼/ブロックリスト、よく使う相手、設定
- `data/cache.json` — 取得済みメールのメタデータキャッシュ

`data/` は `.gitignore` で除外されており、**GitHub には一切アップロードされません**
(このリポジトリは公開のため)。そのため学習リストは PC ごとに独立です。
メール本文はキャッシュせず、表示時に Mail.app から都度取得します。

## 仕組み

```
ブラウザUI (public/)
   ↕ HTTP (localhost:8765)
server.py — API・迷惑判定 (rules.py)・重要度スコアリング
   ↕
mail_store.py — 高速モード: ~/Library/Mail の Envelope Index (SQLite, 読み取り専用)
             — フォールバック: AppleScript (scripts/*.applescript)
   ↕
Mail.app(仕訳・本文取得は常に AppleScript 経由)
```

`docs/` は GitHub Pages 用のデモ(サンプルデータ + `demo.js` によるAPIモック)です。
